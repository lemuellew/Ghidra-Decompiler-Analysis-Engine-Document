\doxysubsection{Detailed Description}
Command to {\bfseries{decompile}} a specific function. 

The command expects 2 string parameters\+: the encoded integer id of the program, and an $<$addr$>$ tag describing the entry point address of the function to decompile. The function follows flow from the entry point up to R\+E\+T\+U\+RN ops or other boundaries of the function. The control-\/flow and data-\/flow structures are built and transformed according to the current configuration of the \mbox{\hyperlink{classArchitecture}{Architecture}} and the active {\itshape root} \mbox{\hyperlink{classAction}{Action}}. Symbols, data-\/types and p-\/code are fetched as needed from the client and cached in the \mbox{\hyperlink{classArchitecture}{Architecture}} object. X\+ML Documents containing source code results, data-\/flow and control-\/flow structures, symbol information, etc., are sent back to the client.
