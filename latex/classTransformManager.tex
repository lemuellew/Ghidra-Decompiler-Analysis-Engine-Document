\doxysubsection{Detailed Description}
Class for splitting larger registers holding smaller logical lanes. 

Given a starting \mbox{\hyperlink{classVarnode}{Varnode}} in the data-\/flow, look for evidence of the \mbox{\hyperlink{classVarnode}{Varnode}} being interpreted as disjoint logical values concatenated together (lanes). If the interpretation is consistent for data-\/flow involving the \mbox{\hyperlink{classVarnode}{Varnode}}, split \mbox{\hyperlink{classVarnode}{Varnode}} and data-\/flow into explicit operations on the lanes.

\mbox{\hyperlink{classVarnode}{Varnode}} marking prevents duplicate \mbox{\hyperlink{classTransformOp}{Transform\+Op}} (and \mbox{\hyperlink{classTransformVar}{Transform\+Var}}) records from getting created, except in the case of a preexisting \mbox{\hyperlink{classPcodeOp}{Pcode\+Op}} with 2 (or more) non-\/constant inputs. Because the op is preexisting the output \mbox{\hyperlink{classVarnode}{Varnode}} doesn\textquotesingle{}t get marked, and the op will be visited for each input. This method determines when the \mbox{\hyperlink{classTransformOp}{Transform\+Op}} object should be created, with the goal of creating it exactly once even though the op is visited more than once. It currently assumes the \mbox{\hyperlink{classPcodeOp}{Pcode\+Op}} is binary, and the slot along which the op is currently visited is passed in, along with the \mbox{\hyperlink{classTransformVar}{Transform\+Var}} for the {\itshape other} input.
