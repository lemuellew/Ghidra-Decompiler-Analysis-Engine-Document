Read a varnode from the given stream. 

The \mbox{\hyperlink{classVarnode}{Varnode}} is selected from the {\itshape current} function. It is specified as a storage location with info about its defining p-\/code in parantheses.
\begin{DoxyItemize}
\item {\ttfamily E\+AX(r0x10000\+:0x65)}
\item {\ttfamily E\+CX(i)}
\item {\ttfamily r0x10001000\+:4(\+:0x96)}
\item {\ttfamily u0x00001100\+:1(\+:0x102)}
\item {\ttfamily \#0x1(0x10205\+:0x27)}
\end{DoxyItemize}

The storage address space is given as the {\itshape short-\/cut} character followed by the address offset. For register spaces, the name of the register can be given instead of the offset. After the offset, a size can be specified with a \textquotesingle{}\+:\textquotesingle{} followed by the size in bytes. If size is not provided and there is no register name, a default word size is assigned based on the address space.

The defining p-\/code op is specified either as\+:
\begin{DoxyItemize}
\item An address and sequence number\+: {\ttfamily E\+AX(r0x10000\+:0x65)}
\item Just a sequence number\+: {\ttfamily E\+AX(\+:0x65)} or
\item An \char`\"{}i\char`\"{} token for inputs\+: {\ttfamily E\+AX(i)}
\end{DoxyItemize}

For a constant \mbox{\hyperlink{classVarnode}{Varnode}}, the storage offset is the actual value of the constant, and the p-\/code address and sequence number must both be present and specify the p-\/code op that {\itshape reads} the constant.
